Intro

Shelve tries to give different approach of exchanging and sharing files across different platform using latest WebRTC protocol.

Main design is that you can quickly share your folder (as shared storage) over network and give other users ability to read or write files. Sharing single file is also possible but files can only be downloaded.

Main features:

  • Cross platform files/folders sharing. Linux, Windows, OSX
  • User can read or write files on shared paths.
  • Exchange is secure and p2p, to allow direct data transmisition
  • Sharing is done in one command and controled from user device

Future Features:

  • Specific permissions and control for each share (NOT DONE)
  • Safe over internet (NOT DONE)

Shelve clients:

  • CLI client
  • Web client - GUI file explorer (no direct sharing possible)

Explore different ways of usage

Shelve is mainly designed to be used with CLI client. Client comes as single static binary which you can easily install and run anywhere.

You can turn any machine also as backup server. This is useful if you need some storage or files always accessible.

You can run it in VM, container (with network configured) and then manage all files with one command.

Webrtc

Clients are using latest WebRTC technology to send and receive files p2p. Webrtc is protocol designed for safe p2p and real time communications.

Install

Installing is easy as it comes as single binary for server and clients.

Usage

Using simple interface allows you to share files/directories, make copies, add files to remote path, explore, search …
